AndroidStudio关联Android源码的正确姿势

发表于 2017-09-11 | 分类于 android

#AndroidStudio查看源码相关配置

最近换了一台机器不知道是安装的时候没搞好还是什么缘故，用AndroidStudio导致查看源码出现了

Sources for ‘Android API 26 Platform’ not found.

Google了一番后，果然还是有前辈遇到了和我一样的问题。虽然如此 但还是想记录下方便以后自己查阅的同时也希望能帮助到遇到同样问题的道友吧

一、当你查看源码的时候出现这种情况：



先不要急忙点击 Download 最好是先去你SDK安装目录下的 sources 文件夹下看下是否已经存在对象API版本的源码 有时候其实已经存在了。

像我这样就是确实没有26的源码




这个时候我有两种选择：

> 1.下载26的源码

> 2.直接使用已存在的源码(23/24/25)

这里先说第二种吧。来到你对应 Module 所在的 build.gradle 文件, 修改compileSdkVersion 和 targetSdkVersion 版本对应存在源码的版本 像我的23/24/25
这里我就改成25的吧

如果你修改之后还是查看不了,这个时候你就需要去修改下配置文件了,文件目录在

windows下：系统盘:\Users\username.AndroidStudio2.3\config\options

Mac下：~/Library/Preferences/.AndroidStudio2.3/config/options

找到jdk.table.xml,如果你有 Everything 可以直接搜索, Mac用户可以直接用 Spotlight搜索,然后打开修改对应的版本关联信息。

最后记得重启AS
